# Kid-Simple Guide: Evo Studio Builder

## What is this?
It is a robot builder.

You are not coding the whole robot by hand.
You are choosing what blocks the robot gets.

## The blocks

### Brain
What is this thing?
Example: “Flutter App Builder Evo LM.”

### References
What should it read?
Example: files, PromptBase records, screenshots, API docs.

### Tools
What can it use?
Example: GitHub, Flutter, image generator, file search.

### Rules
What can it never do?
Example: never leak secrets, never claim deployment without proof.

### Approval
When must it ask you?
Example: before writing GitHub code, sending messages, deploying, or spending money.

### Tests
How do we check it?
Example: “Does it create a screen map?” “Does it ask before GitHub writes?”

### Proof
What receipt shows it happened?
Example: build log, screenshot, PR link, test report.

### Publish
Where does it go?
Example: PromptHouse Studio, AI chat, Antigravity, VS Code, Flutter app.

## The simple build flow
Pick type → name it → add references → pick tools → set rules → test it → export it.
