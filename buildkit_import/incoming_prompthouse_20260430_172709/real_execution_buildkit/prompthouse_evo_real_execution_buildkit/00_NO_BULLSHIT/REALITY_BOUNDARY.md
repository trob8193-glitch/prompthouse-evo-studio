# Reality Boundary

The package can be built and validated here as files. It cannot honestly be called deployed or production-ready until it runs in your environment with real keys and tests.

## Forbidden claims

- omnipotent
- absolute perfection
- nothing left to build
- self-healing without logs/tests/repair code
- production-ready without deployment proof
- API-connected without configured keys and live successful calls

## Required truth states

- known
- inferred
- built
- verified
- blocked
- broken
- recommended

## Feature completion rule

A feature is complete only when:
- code exists
- route exists
- endpoint exists
- persistence exists if needed
- tests exist
- errors are handled
- proof card exists
- rollback is defined
- runtime proof exists
