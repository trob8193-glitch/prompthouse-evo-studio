# PH Evo ForgeRail + Evo LM + Studio Extension Foundry v1.0

Built: 2026-04-30

## Mission
Build ForgeRail 5X plus Evo LM plus a no-bullshit extension system so PromptHouse bots can create studio extensions using PromptHouse logic.

## What this is
A deployable architecture/package for:
- ForgeRail 5X self-build accelerator
- Evo LM, a PromptHouse-native orchestration layer
- PromptLink bridge protocol
- 10 senior prompt architecture engineering dev bots
- Studio Extension Foundry
- ExtensionCapsule schema
- IDE/mobile/backend/provider extension scaffolds
- proof-first install/build/test/ship gates

## Truth boundary
Evo LM is not a clone of ChatGPT, Claude, Gemini, Codex, or any proprietary model.
It is a PromptHouse orchestration layer that can route to external providers through PromptLink using real credentials, scopes, budgets, and permission.
Studio extensions are built as code/spec packages until actually installed and verified.
No fake deployment. No fake store submission. No fake external tool control.

## Core flow
User command → Evo LM → 11 PromptHouse dev bots → senior prompt architecture team → ForgeCapsule/ExtensionCapsule → PromptLink/ForgeRail → proof receipts → repair loop.
