# Studio Extension Foundry

## Mission
Let PromptHouse bots build studio extensions using PromptHouse logic.

## Extension types
- Flutter UI panel
- VS Code command/extension
- Android Studio/IntelliJ plugin scaffold
- Antigravity mission pack
- ForgeRail rail
- PromptLink provider adapter
- PromptBase pack
- Proof Deck artifact type
- backend module
- AI-chat kernel pack
- GitHub Action
- QA/device test pack

## Build flow
extension request → Prompt Auth → ExtensionCapsule → owner/risk classification → code scaffold → tests → proof receipt → install instructions → optional install approval

## No-bullshit boundary
A bot may build an extension package and install instructions. It may not claim the extension is installed, published, marketplace-approved, or connected unless proof exists.

## Bot ownership
- Evo: product/mission approval
- Dev: code scaffold
- Builder: package/artifact assembly
- Verifier: tests and proof
- Boundary: permissions and risk
- Ledger: receipts and versioning
- Memory: PromptBase and registry
- Conductor: route and dependency order
- Companion: user onboarding
- Heartbeat: status and health checks
- Sovereignty: owner approval and command authority
